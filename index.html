Module 6: Logic, Conditionals, and Loops

Logic, Conditionals, and Loops are some of the most important concepts in all of Software Development. When we work with Loops, we very often are also working with Arrays (or another Data Structure). One of the most common things to do with a loop is to access every element of an Array to do something. Often we want to test a condition (a test) to see determine what we will do next in our program.

Conditionals:

  • if statements
    • Test that something is true.
    • Underneath the condition (the test that something is true) there is a block of code that will only execute if that condition (test) is true.
    • if statements can exist on their own, so you may use as many as you like or need.
  • else if statements
    • Test that something else is true.
    • Only execute when the if statement is false, but another condition is true.
    • Cannot exist on their own.
    • else if statements always come after an if statement, but before the else statement.
  • else statements
    • Do not test any condition, so they only use the else keyword.
    • Cannot exist on their own.
    • Every else statement must follow an if statement.
    • Followed by a block of code that will only execute when the condition (test) of the if statement was false.


Loops
    Types of For Loops:
    • Standard For Loops
      • There are three statements in a for loop.
        1. An index variable declaration that is used to keep track of your place in a loop.
        2. A condition that is used to determine when the loop will execute.
        3. Incrementing (increasing) your variable by an appropriate amount.
      • The most flexible kind of loop, since you can loop forwards or backwards.
      • Exists in all programming languages
    • For/Of Loops
      • Designed as an easy way to loop through anything that can be iterated (string, array, list, etc).
      • Requires you to create an index variable inside the loop declaration.
      • Handles looping for you.
      • Easy syntax, but less flexible than a standard for loop.
      • Can only loop in one direction (start to end).
      • Can only iterate by one.
      • Many languages have a version of this, but not all languages support for/of loops specifically.
    • For/In Loops
      • Designed specifically to loop through the properties of an Object.
      • Excellent for working with objects, but not well suited to looping over other things.
      • Requires you to declare an index variable (like for/of loops do) to keep track of your place in the loop.
    Types of While Loops:
    • Standard While Loops
      • Continues executing a block of code while a condition is true.
      • Requires you to declare an index variable outside of the loop and its declaration.
      • Requires you to increase the index inside the block of code.
      • Less forgiving than for loops, since you must iterate your index variable.
      • Forgetting to iterate your index variable will cause an infinite loop that will crash your browser.
    • Do/While Loops
      • Works exactly like a Standard While Loop, but it will always execute a single time regardless of whether or not the condition is true.
      • Declared using the do keyword.
      • The condition only comes after the block of code to execute.
